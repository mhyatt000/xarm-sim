# Sim Lift Dataset — Training Handoff

Synthetic xArm7 block-lift episodes as Foxglove protobuf MCAP, generated by this repo
and format-identical to the real recordings under `/data/store/mcaps/single/lift`.
Intended consumer: crossformer (`mhyatt000/crossformer`; its loader derives
`action = proprio.copy()`, so complete proprio is sufficient supervision).

## Where the data is

| Batch | Path | Notes |
| --- | --- | --- |
| pilot_v2 | `outputs/sim_mcap/pilot_v2/` | 10 episodes, style-matched policy |
| current generation root | `/data/store/griffen_sim_mcaps/` | large-disk destination for new sim MCAP batches |
| full batch | `outputs/sim_mcap/batch_v1/` | historical local batch, generated after CP-B sign-off |

Every batch directory contains a `manifest.json` (git SHA, full config, splat md5,
per-episode seeds and stats) — any episode can be regenerated bit-for-bit.

Current large-disk generation root:

```bash
/data/store/griffen_sim_mcaps/<name>/
```

## What is calibrated vs what is guessed

**Calibrated / measured (trust these):**
- `low`/`side` camera intrinsics+extrinsics: from `/data/store/opencv_calibrated`
  (today's rig, robot-base frame). Sim cameras render at exactly these poses.
- Table plane z=0 and its footprint: measured by back-projecting the calibration photos.
- Splat (background) placement: solved against the ZED fused point cloud; verified by
  render-vs-photo blends. Robot silhouette matches real photos at **median 1.9 px**
  across all 24 calibration configs (`outputs/blink_test/`).
- Proprio/labels: generated directly in the robot frame; never touch the splat.
- Demonstration style: start height, lift height, grasp timing, tempo variation tuned to
  match the real episodes' profiles (`outputs/batch_report/report_pilot_v2.png`).

**Approximate / guessed (know these):**
- Wrist camera mount: no calibration exists (the robot is never in its own view). The
  mount was hand-fit and then validated against real May wrist frames at matched joint
  angles (~40 px agreement mid-grasp) — good, not exact.
- Camera intrinsics use the calibration's approximate model (fx=fy=515, centered
  principal point, no distortion). This is consistent end-to-end (the extrinsics were
  solved under the same model), but a checkerboard intrinsic calibration would lift
  accuracy across the board if ever needed.
- Lighting/appearance: one directional light tuned by eye; the splat covers the
  background but the robot/cube/table are mesh renders.
- Recorded gripper close depth reads norm≈0.32 vs real ≈0.37 (rigid sim fingers need
  extra squeeze to hold the cube).

## Evaluation caveats (important)

- The 409 real episodes were recorded on the **old** camera rig. Sim matches **today's**
  calibrated rig. For the two static cameras, don't offline-eval a sim-trained,
  image-conditioned policy against the May files and expect meaningful numbers — the
  viewpoints moved. Meaningful evals: live rollout on the current rig, or record a
  handful of fresh real episodes on the current rig for held-out offline eval.
- The wrist camera predates the rig change, so wrist-view comparisons against May data
  are fair.

## Regenerating / scaling

```bash
# N episodes, photoreal, style-matched (slab table is the default; do not use table-mode plane)
uv run python scripts/generate_lift_dataset.py --n-episodes N \
    --env.render-backend nyx --out-dir /data/store/griffen_sim_mcaps/<name> --seed <fresh>
# verify any batch:
uv run python scripts/compare_batches.py --sim-dir /data/store/griffen_sim_mcaps/<name>
# re-check camera/scene accuracy after any recalibration or rescan:
uv run python scripts/blink_test.py            # then re-solve with align_ransac.py if needed
```

Cost: ~70 s/episode (Nyx, RTX 5090) ≈ 2 h per 100 episodes.
